# ระบบ 2 ภาษา (i18n)

โปรเจกต WP Property ใช้ **2 ภาษาเสมอ**: ไทย (หลัก) และ English

## Tech stack

- [`@nuxtjs/i18n`](https://i18n.nuxtjs.org/) v10
- ไฟล์แปล: `i18n/locales/th.json`, `i18n/locales/en.json`

## URL

| ภาษา | ตัวอย่าง |
|------|----------|
| ไทย (default) | `/`, `/#services` |
| English | `/en`, `/en#services` |

- Strategy: `prefix_except_default` — ไทยไม่มี prefix, อังกฤษใช้ `/en`
- **Admin** (`/admin/**`) ไม่แปล route — ใช้ภาษาไทยใน UI หลังบ้านตามเดิม

## กฎเมื่อเพิ่มหน้า / component ใหม่

1. **ห้าม hardcode ข้อความภาษา** ใน template — ใช้ `$t('key')` หรือ `t('key')` เสมอ
2. **เพิ่ม key ทั้ง 2 ไฟล์** `i18n/locales/th.json` และ `i18n/locales/en.json` พร้อมกัน
3. **ลิงก์ภายในเว็บ public** ใช้ `localePath()` / `<NuxtLink :to="localePath('/about')">`
4. **สลับภาษา** ใช้ `switchLocalePath()` — ไม่ hardcode `/en/...`
5. **ข้อมูลที่ไม่แปล** (รูป, ราคา, ID) เก็บใน `app/data/` — ข้อความอ้าง key ใน locale

## โครงสร้าง key

```
nav.*           — เมนู header / footer
common.*        — คำใช้ซ้ำ (ค้นหา, ดูทั้งหมด, ขาย, เช่า)
header.*        — header เฉพาะ (alt, aria)
footer.*        — footer
home.*          — หน้าแรก (hero, services, search, sections)
```

ตัวอย่าง array content ใน component:

```ts
// app/data/home-content.ts — ไม่มีข้อความภาษา
export const services = [
  { key: 'debt', image: '...' },
  { key: 'rentBuy', image: '...' },
]

// template
{{ t(`home.services.${service.key}.title`) }}
```

## Composables / components

| ไฟล์ | 用途 |
|------|------|
| `useSiteNav()` | เมนู public พร้อม `localePath` และ hash |
| `SiteLocaleSwitcher.vue` | ปุ่มสลับ TH / EN |

## ตัวอย่างโค้ด

```vue
<script setup lang="ts">
const { t } = useI18n()
const localePath = useLocalePath()
</script>

<template>
  <NuxtLink :to="localePath('/')">{{ t('nav.home') }}</NuxtLink>
</template>
```

```vue
<!-- สลับภาษา -->
<script setup lang="ts">
const switchLocalePath = useSwitchLocalePath()
</script>

<NuxtLink :to="switchLocalePath('en')">English</NuxtLink>
```

## หน้า admin

Route ใต้ `/admin/**` ตั้ง `i18n: false` ผ่าน hook ใน `nuxt.config.ts` — ไม่ต้องใส่ `$t()` ใน admin จนกว่าจะต้องการแปลหลังบ้าน

## ข้อควรระวัง

- ใน vue-i18n **ห้ามใช้ `@` ตรงๆ** ในข้อความ (เช่น Line ID) — ใช้ `{'@'}` แทน เช่น `"Line: {'@'}wpproperty"`

## Cookie / browser language

- Cookie: `wp_i18n`
- เปิด `/` ครั้งแรกอาจ redirect ตาม browser language (ถ้าเป็น en)
